### Step 2: Daily Development Pattern
```bash
# Each morning (10 min)
cat week-X-Y/CONTEXT.md         # Refresh weekly goals
cat week-X-Y/PROMPTS.md         # See today's tasks

# During day (15-20 min per task)
# 1. Pick a prompt
# 2. Open Cursor IDE (Cmd+K)
# 3. Paste prompt
# 4. Get generated code
# 5. Copy to file
# 6. Verify: python -c "import..."
# 7. Run tests: pytest
# 8. Commit: git commit -m "feat: ..."

# Each evening (5 min)
make test                        # All passing?
pytest --cov=backend             # 85%+ coverage?
git log --oneline -5             # What did I build?
```

### Step 3: Weekly Review (30 min Friday)
```bash
# Full code audit
make lint
make type-check
make test
pytest --cov

# Generate coverage report
pytest --cov=backend --cov-report=html
open htmlcov/index.html

# Review progress
git log --oneline -15            # What got done this week?

# Plan next week
cat week-X+2-Y+2/CONTEXT.md     # What's coming?
```

## 💡 Key Concepts

### Atomic Tasks
Each prompt/task = **15 minutes of work**
- Not "build the entire database" (8 hours)
- But "create the ScanRun model" (15 min)
- Each task = 1 commit

### Quality Gates
Every code must pass before moving forward:
1. ✅ No import errors
2. ✅ Type hints 100%
3. ✅ Tests pass (all of them)
4. ✅ Code formatted (black, flake8)
5. ✅ Coverage >85%
6. ✅ Docstrings present

### Multi-Agent Coordination
```
Agent 1 (Cursor):  Writing models.py
Agent 2 (VS Code): Writing tests simultaneously
Agent 3 (Claude):  Designing next feature
Result: 3x faster!
```

### Vibe Coding
- Flow-based development (relax, let AI handle details)
- Quality-first (every commit is production-ready)
- Professional output (no "quick fix" debt)
